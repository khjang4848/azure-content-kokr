# Azure SQL 데이터베이스의 보안 유지 및 보호

## 앱의 데이터를 보안 및 보호 유지

Azure SQL 데이터베이스는 보안, 규정 준수 및 고성능등 높은 수준의 가용성을 활용하는 클라우드에서 수많은 업무에 중요한 데이터베이스를 지원합니다. 데이터 보안이 소규모 기업에서 큰 엔터프라이즈까지 모든 비즈니스에서 점점 중요해지면서 SQL 데이터베이스는 모든 사용자를 지원합니다.

![helps-secures-and-protects](./media/sql-database-helps-secures-and-protects/sql-database-helps-secures-and-protects.png)

## Microsoft가 도와 드리겠습니다!

보통 이전 백업에 액세스하거나 재해에서 복구할 필요가 없기를 바랍니다. 하지만 이 경우에 SQL 데이터베이스가 도움이 됩니다. 99.99%의 가용성을 지닌 자동 백업, 지정 시간 복원, 지역에서 기본 제공 복제 및 장애 조치 옵션으로 앱은 안전합니다.

[설명서 읽기](sql-database-business-continuity.md)

## 가장 큰 규정 준수 포트폴리오

SQL 데이터베이스에는 업계에서 가장 큰 규정 준수 포트폴리오가 있습니다. 따라서 HIPPA 규정 준수나 FERPA 또는 싱가포르 MTCS 수준 3이 필요한 경우라도 충분히 사용할 수 있습니다.

[설명서 읽기](https://www.microsoft.com/TrustCenter/Compliance/default.aspx)

## 보호 계층

SQL 데이터베이스는 여러 단계의 데이터 보호를 제공합니다. 휴식하는 동안 동작 중이거나 사용 중인 데이터를 암호화하고 데이터베이스 또는 응용 프로그램에 대해 권한이 있는 사용자만을 인증하며 데이터의 적절한 하위 집합에 대한 사용자 액세스를 제한하여 데이터를 보호합니다. 또한 SQL 데이터베이스는 활동을 연속 모니터링하고 감사하여 위반이 발생할 경우 잠재적인 위협을 검색하고 중요한 이벤트의 레코드를 제공합니다. 이러한 기능은 개발자 생산성 또는 고객 환경을 손상하지 않고 기능을 신속하게 구현하고 보안 위험을 줄일 수 있는 기능으로 부하가 분산됩니다.

[설명서 읽기](http://go.microsoft.com/fwlink/?LinkID=787593)

SQL 데이터베이스에서 제공해야 하는 다른 항목을 보려면 홈 페이지로 이동합니다. [확인해 보세요.](https://azure.microsoft.com/services/sql-database/)
